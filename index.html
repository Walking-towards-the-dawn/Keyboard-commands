Commands

Git

    Getting & Creating Projects

  • $ git init
    Ініціалізація локального сховища Git
  • $ git clone [URL]
    Створити локальну копію віддаленого репозиторію

    • Basic Snapshotting

  • $ git status
    Перевірити статус
  • $ git add [file-name.txt]
    Додати файл до робочої області
  • $ git add -A
    Додати всі нові та змінені файли до робочої області
  • $ git commit -m "[commit message]"
    Зафіксувати зміни
  • $ git rm -r [file-name.txt]
    Видалення файлу (або папки)

    • Branching & Merging

  • $ git branch
    Список гілок (зірочка позначає поточну гілку)
  • $ git branch -a
    Список усих гілок(локальних і віддалених)
  • $ git branch [branch name]
    Створити нову гілку
  • $ git branch -d [branch name]
    Видалити гілку
  • $ git push origin --delete [branch name]
    Видалити віддалену гілку
  • $ git checkout -b [branch name]
    Створити нову гілку та перейти до неї
  • $ git checkout -b [branch name] origin/[branch name]
    Клонувати віддалену гілку та перейти до неї
  • $ git branch -m [old branch name] [new branch name]
    Перейменувати локальну гілку
  • $ git checkout [branch name]
    Перейти до гілки
  • $ git checkout -
    Перемикає на попередню гілку, на якій ви перебували
  • $ git checkout -- [file-name.txt]
    Скасувати зміни у файлі
  • $ git merge [branch name]
    Об’єднати гілку в активну гілку
  • $ git merge [source branch] [target branch]
    Об’єднати гілку в цільову гілку
  • $ git stash
    Збереже всі зміни в робочому каталозі і індексі, і автоматично призначить їм дефолтний опис
  • $ git stash clear
    Видалити усі приховані записи

    • Sharing & Updating Projects

  • $ git push origin [branch name]
    Надіслати гілку до свого віддаленого сховища
  • $ git push -u origin [branch name]
    Надіслати зміни до віддаленого сховища (і запам’ятати гілку)
  • $ git push
    Надіслати зміни до віддаленого сховища (збережена гілка)
  • $ git push origin --delete [branch name]
    Видалити віддалену гілку
  • $ git pull
    Оновити локальне сховище до останнього коміту
  • $ git pull origin [branch name]
    Отримати зміни з віддаленого сховища
  • $ git remote add origin [URL]
    Додайти віддалене сховище
  • $ git remote set-url origin [URL]
    Для зміни посилання на новий сервер або для переключення на інший URL репозиторій

    • Inspection & Comparison

  • $ git log
    Переглянути зміни
  • $ git log --summary
    Переглянути зміни (детально)
  • $ git log --oneline
    Переглянути зміни (коротко)
  • $ git diff [source branch] [target branch]
    Переглянути зміни перед об’єднанням

Fedora Linux

  • ~ cd
    Перейти в домашню папку
  • ~ cd -
    Переміститися до попередньої папку
  • ~ cd ..
    Переміститися на один каталог вгору
  • ~ clear / Ctrl + L
    Очищає вікно терміналу від усих повідомлень
  • ~ history
    Відображає всі нещодавно введені вами команди
  • ~ whatis ім`я_пакету
    Відображає короткий опис будь-якої програми
  • ~ whereis ім`я_пакету
    Відображає розташування файлу потрібної програми
  • ~ sudo dnf install ім`я_пакету
    Встановити пакет
  • ~ sudo dnf update / sudo dnf check-update
    Оновити інформацію про пакети
  • ~ sudo dnf upgrade
    Оновити всі пакети до найсвіжіших (виконувати після dnf update)
  • ~ sudo dnf remove ім`я_пакету
    Видалити непотрібний пакет
  • ~ sudo dnf purge ім`я_пакету
    Видалити непотрібний пакет із усіма залежностями
  • ~ sudo dnf autoremove
    Видалити всі непотрібні залежності
  • ~ kill / kill PID_процесу
    Примусове завершення процесів. PID процесу можна дізнатися, ввівши top
  • ~ xkill
    Команда для завершення процесів. Після введення команди потрібно клацнути по тому вікну, яке потрібно закрити
  • ~ killall
    Завершує процеси з певним іменем. Наприклад, killall firefox
  • ~ cat
    Використовується для виведення вмісту файлу командного рядка
  • ~ cat > ім`я_файлу
    Створює новий файл
  • ~ cat ім`я_файлу(1) ім`я_файлу(2) > ім`я_файлу(3)
    Поєднує два файли (1 і 2) і збереже їх вміст у новому файлі (3)
  • ~ chmod
    Дозволяє змінювати права доступу до файлу. Може стати в нагоді, якщо ви хочете внести зміни до системного файлу
  • ~ chown
    Змінює власника файлу чи каталогу. Слід виконувати із правами суперкористувача. Наприклад, chown user:group ваш_файл змінить власника та групу файлу на задані
  • ~ file
    Виводить інформацію про вказаний файл
  • ~ rename
    Перейменовує один або декілька файлів
  • ~ mkdir ім'я_каталогу
    Для створення нового каталогу
  • ~ mkdir ім`я_каталогу(1)/ім`я_каталогу(2)
    Cтворює новий каталог усередині іншого каталогу
  • ~ mkdir -p ім`я_каталогу(1)/ім`я_каталогу(2)/ім`я_каталогу(3)
    Створює каталог між двома існуючими каталогами
  • ~ rmdir
    Використовується для видалення файлів
  • ~ cp
    Для копіювання файлів із поточного каталогу до іншого каталогу. Наприклад, команда cp scenery.jpg/home/username/Pictures створить копію scenery.jpg (з поточного каталогу) до каталогу Pictures
  • ~ mv
    Переміщує файл із однієї папки до іншої. Аргументи в mv схожі з аргументами команди cp. Наприклад: mv file.txt/home/username/Documents
  • ~ grep
    Пошук текстових файлів, які містять певні рядки
  • ~ locate
    Пошук файлів та папок, чиї назви відповідають запиту, та відображеє їх шлях у файловій системі
  • ~ ls
    Використовується для перегляду вмісту каталогу
  • ~ la -la
    Використовується для перегляду всього вмісту каталогу
  • ~ ls -R
    Виводить список усіх файлів у підкаталогах
  • ~ ls -a
    Показує приховані файли
  • ~ ls -al
    Виведить список файлів та каталогів з детальною інформацією
  • ~ touch
    Дозволяє створити новий порожній файл. Наприклад , touch /home/username/Documents/Web.html , щоб створити файл HTML з назвою Web у каталозі Documents
